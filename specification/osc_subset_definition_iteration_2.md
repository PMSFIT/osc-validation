# Subset Definition: Release Subset (Iteration 2)

[Back to ASAM OpenSCENARIO XML Subset Definition](./osc_subset_definition.md)

The release subset extends Iteration 1 with the OpenSCENARIO XML features that are currently covered by the validation suite.
Iteration 1 remains the historical minimal subset for basic OSI SensorView trace replay.
This subset still targets OpenSCENARIO XML 1.3.0 scenario definitions.
Where validation cases reference clarified behavior from OpenSCENARIO XML 1.4.0, that clarification is documented as a validation expectation for the covered behavior and not as a general expansion to all OpenSCENARIO XML 1.4.0 features.

## General Constraints

All optional elements and attributes not explicitly included in this specification remain excluded.
Only explicitly included action types, trigger and condition types, position types, timing behavior, and storyboard structures are included.
The vehicle-focused constraints from Iteration 1 remain unless explicitly expanded in this section.

## Included Features

### Storyboard and Sequencing

The storyboard may use multiple sequential storyboard elements where covered by the split trajectory validation cases.
Sequential trajectory fragments are validated at the following storyboard levels:

- `Event`
- `Maneuver`
- `ManeuverGroup`
- `Act`
- `Story`

Sequencing between split trajectory fragments may use `StoryboardElementStateCondition` with `state="completeState"`.
The storyboard stop trigger may use `SimulationTimeCondition` to terminate the scenario at a defined simulation time.

### Scenario Objects

Only scenario objects of type `vehicle` are included.
Only vehicles of category `car` are included.
Vehicle dimensions and bounding box dimensions are validated against OSI reference traces for the covered vehicle objects.
The remaining vehicle constraints from Iteration 1, including front and rear axle requirements, continue to apply.

### Init Actions

The release subset includes the following `Init` actions:

- Private `TeleportAction` with `WorldPosition`
- `AddEntityAction` with `WorldPosition`
- Private `SpeedAction` with an absolute speed target

### Trajectory Actions, Timing, and Coordinate Systems

The release subset includes `FollowTrajectoryAction` with:

- `Polyline` trajectory shape
- Timed `Vertex` elements
- `WorldPosition` positions
- `followingMode="position"`
- Absolute time reference cases for the current trajectory time and future trajectory time validation cases
- Relative time reference cases generated by the OSI trace replay and split trajectory validation cases

Trajectory interpolation is validated for timed polyline trajectories in position mode.
The validation suite includes the OpenSCENARIO XML 1.4.0 constant-acceleration interpolation expectation and also permits the linear position interpolation behavior used for older-version compatibility checks.
Versions before OpenSCENARIO XML 1.4.0 did not require a specific interpolation method.

### Triggers and Conditions

The release subset includes event `StartTrigger` conditions for activating validated trajectory behavior.
The following condition types and attributes are included:

- `SimulationTimeCondition`
- `TraveledDistanceCondition`
- `SpeedCondition` with `conditionEdge` values `rising`, `falling`, and `none`
- `DistanceCondition` to `WorldPosition` with `relativeDistanceType="euclidianDistance"`
- `DistanceCondition` to `WorldPosition` with `relativeDistanceType="longitudinal"`
- `TimeToCollisionCondition` to target `WorldPosition`
- `Condition` `delay` for speed and longitudinal distance trigger cases

Unless stated above, trigger behavior remains outside the release subset.
